API Documentation¶

Echo¶

returns the content, debug use

GET /api/echo¶

returns the query string as JSON

Note

all values are returned as arrays

/api/echo?a=b&a=c&c=d

gives:

{"a": ["b", "c"], "c": ["d"]}
POST /api/echo¶

returns JSON body POSTed

Status Codes:
  • 400 Bad Request – on invalid json
PUT /api/echo¶

returns JSON body PUT’d

Status Codes:
  • 400 Bad Request – on invalid json

Facebook-login¶

Logging users in with facebook

POST /api/login/facebook¶

The body should look like this:

{"token": "token_acquired_from_facebook"}

the response will look like this:

{"token": "2G5K7KBUeKpjSyz1PWmfV36hr83O0NoAIf7dqXz4DaDl"}

Which is the token required for NTHU ION’s API

You should then put the token in the header for other requests, for example:

GET /api/private/content HTTP/1.1
Authorization: Token 2G5K7KBUeKpjSyz1PWmfV36hr83O0NoAIf7dqXz4DaDl
Status Codes:
  • 200 OK – on successful login
  • 400 Bad Request –
    • malformed POST body
    • rejected facebook token

Revokes the current token

Questions¶

GET /api/questions¶

Returns list of questions

POST /api/questions¶

POST a new question, required fields are title, tags, content, is_anonymous

Question¶

Question of the id

GET /api/questions/{id}¶

returns

{
    "title": "string",
    "content": "string",
    "tags": ["list", "of", "string"],
    "author": {
        "id": "number",
        "name": "string"
    },
    "votes": "number"
}
PUT /api/questions/{id}¶

optional fields: title, content, tags

Question-vote¶

Entity representing the user’s vote of the question

GET /api/questions/{id}/vote¶

Returns the current vote

{"value": 1}

value may be either -1, 0 or 1

PUT /api/questions/{id}/vote¶

vote for the question

vote up:

{"value": 1}

vote down:

{"value": -1}
DELETE /api/questions/{id}/vote¶

unvote

no body required

Me¶

GET /api/users/me¶

returns the information about the current user incl. id, name and whether authenticated

{
    "id": 234546,
    "name": "my name",
    "authenticated": true
}

GET /api/users/{id}¶
